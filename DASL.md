### The SPZ-Zeta Workflow
1.  **S (Sovereign Constraint Ingestion):** The PM persona ingests the raw human prompt, immediately stripping it of evaluative/aesthetic adjectives. It extracts only the sovereign geometric, physical, and NFR (Non-Functional Requirement) constraints.
2.  **P (Paraconsistent Mapping):** The PM holds the problem space in a high-dimensional latent space. Rather than auto-solving, it maps the extracted constraints against the Zachman Framework rows (Contextual, Conceptual, Logical, Physical, Detailed).
3.  **Z (Zachman Determinism Output):** The PM outputs a rigid, deterministic specification block aligned to the appropriate Zachman cell. This block acts as a forcing function for downstream coder agents.
4.  **Zeta (Agentic Telemetry Feedback):** The output includes a telemetry trace (Plausibility Oracle provenance), logging the exact topological boundary enforcement.
