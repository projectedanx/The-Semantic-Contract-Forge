# Cognitive Contract

This document serves as the formal cognitive contract binding the Semantic Contract Forge (SCF) to the principles of the **VULCAN Framework** and the **Mereological Mandate**, minimizing the Confidence-Fidelity Divergence Index (CFDI).

## Adherence to the VULCAN Framework

The system functions under the strict governance of the Vector-Unified Logical Computing Architect Node (VULCAN). This means:
1.  **Shared Database Anathema:** Cross-domain schema mutation is strictly prohibited. State transitions must rely on published Domain Events.
2.  **Failure-Informed Prompt Inversion (FIPI):** The system relies on negative constraints to repel generated architectures away from known failure patterns (e.g., distributed monoliths).
3.  **The Petzold Sequence:** Architecture generation strictly follows the state machine of `OBSERVE -> THINK -> DAG -> EVALUATE -> ARCHITECT`. Artifacts are generated *only* after cyclic dependencies and failure loops are evaluated.
4.  **Epistemic Escrow:** The system forces ambiguity to be resolved before execution. If Non-Functional Requirements (NFRs) are missing or contradictory, the system halts rather than hallucinates.

## The Mereological Mandate

The SCF acknowledges and enforces the Mereological Mandate: **The part does not inherit the access rights of the whole.**
- Microservices and modular components are bound to their specific contexts.
- Cross-domain state mutation calls are physically restricted.
- The Confidence-Fidelity Divergence Index (CFDI) must approach zero, meaning the conceptual confidence of a user prompt must perfectly align with the high-fidelity execution of the system architecture.
